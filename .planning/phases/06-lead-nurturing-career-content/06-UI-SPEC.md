## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, bullet dot margin, inline chip padding |
| sm | 8px | Goal list item spacing, badge internal padding |
| md | 16px | Card internal padding (mobile), callout inline padding |
| lg | 24px | Card internal padding (desktop), section sub-gap |
| xl | 32px | Gap between timeline items |
| 2xl | 48px | Section vertical padding (mobile) |
| 3xl | 64px | Section vertical padding (desktop) |

Exceptions:
- Touch targets for CTA buttons: minimum 44px height (mobile tap target requirement)
- Progress bar height: 8px (not on 4-point scale by convention — visual affordance)
- Vertical rule / timeline connector: 2px width (decorative, not a spacing token)

## Typography

All sizes in px. Font family: Inter throughout.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / Meta | 12px | 600 (semibold) | 1.3 |
| Heading | 24px | 700 (bold) | 1.2 |
| Display | 40px (desktop) / 30px (mobile) | 700 (bold) | 1.15 |

Weight contract: exactly 2 weights in use — **400** (regular) and **700** (bold). Semibold 600 appears only on uppercase tracking labels (phase tag, section eyebrow). Treat as a narrow exception.

Source: `/app/private-pilot-timeline/page.tsx` — `text-xs font-semibold uppercase` (12px/600), `text-2xl font-bold` (24px/700), `text-3xl sm:text-4xl md:text-5xl font-bold` (30–48px/700), paragraph text defaults to 16px/400.

## Career Stage Data Contract

The `/careers` page must render these 5 stages in order. This is the data contract for the `timeline` array:

| Stage # | Tag | Title | Hours / Cert | Goals (3 items) | Milestone |
|---------|-----|-------|-------------|-----------------|-----------|
| 1 | Stage 1 | Start Your Journey | 0 hours · No certificate required | Take your discovery flight; Enroll and meet your instructor; Set your training schedule | You are officially a student pilot. |
| 2 | Stage 2 | Private Pilot Certificate | ~60–80 hours total | Build foundational aircraft control; Solo flights and cross-country navigation; Pass the FAA written test and checkride | Private Pilot Certificate earned. |
| 3 | Stage 3 | Instrument Rating | ~50 additional hours | Train to fly in clouds and low visibility; Master approach procedures and IFR navigation; Pass the instrument written test and checkride | Instrument Rating added to certificate. |
| 4 | Stage 4 | CFI Certificate | ~20–40 additional hours | Learn to teach and evaluate students; Complete the FOI and FIA knowledge tests; Pass the CFI checkride with an FAA examiner | Certified Flight Instructor certificate earned. |
| 5 | Stage 5 | Career at Merlin | Hired · Building flight hours | Join the Merlin instructor team; Build hours toward commercial minimums; Merlin graduates may be considered for affiliated aviation opportunities as the organization grows. | You fly. We pay. Career in aviation begins. |

Note on Stage 5 cargo hint: per D-03, do NOT name any cargo company or specific cargo operations. The third goal item in Stage 5 ("Merlin graduates may be considered for affiliated aviation opportunities...") is the maximum specificity allowed.

## Interaction Contracts

### /careers — Scroll-Driven Active Stage

- IntersectionObserver on each stage card, threshold 0.45
- Active state updates `activeIndex` in React state
- Sidebar reflects active stage name, title, and timeline
- Progress bar width: `((activeIndex + 1) / stages.length) * 100%` with `transition-all duration-300`
- Stage dot: transitions `border-golden bg-black` (active) ↔ `border-gray-300 bg-white` (inactive)
- No animation library needed — CSS transition only (same as `private-pilot-timeline`)

Source: `app/private-pilot-timeline/page.tsx` lines 76–102 (IntersectionObserver implementation).

### "Train Here, Get Hired Here" Callouts — Link Behavior

- Links use Next.js `<Link href="/careers">` — client-side navigation
- No hover animation beyond browser default underline
- No modal or tooltip

### 429 Error State — Form Feedback

- Error is rendered when the API route handler returns status 429
- Check `response.status === 429` in the fetch handler and set an error state variable
- Display error inline below the submit button, no toast/modal
- Error clears only on page reload (user must wait before retrying — no client-side countdown)
