## Spacing Scale

| Token | Value | Usage |

| sm | 8px | Compact row padding, badge padding |
| md | 16px | Default element spacing, card inner padding |

- Touch targets on mobile: minimum 44px height for all interactive controls (buttons, nav links, form rows) — source: WCAG 2.5.5 / iOS HIG

## Typography

All text uses Inter. Weights are limited to two: regular (400) and semibold (600). The admin zone establishes this pattern; the CFI portal inherits it.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Table cells, form labels, description text |
| Label | 12px | 600 | 1.4 | Section eyebrows (uppercase + tracking), badge labels, table column headers |
| Heading | 20px | 600 | 1.3 | Card titles, section headings within the portal |
| Display | 30px | 700 | 1.2 | Page-level `<h1>` (e.g., "CFI Workspace") — matches `text-3xl font-bold` in AdminPageShell |

Note: 12px labels MUST use `uppercase tracking-[0.2em]` to match the eyebrow pattern established in `AdminTopNav` and `AdminPageShell` (`text-xs uppercase tracking-[0.2em]`).

## Color

The project brand accent is `#FFBF00` (golden). The admin zone uses a warm amber derivative palette for nav chrome. The CFI portal follows the same warm-amber chrome to signal "operator zone" (distinct from the public-facing site, which is dark/dramatic). Exact CSS variable resolution from `globals.css`:

| Role | Value | CSS Var | Usage |
|------|-------|---------|-------|
| Dominant (60%) | `#FFFFFF` | `--background: 0 0% 100%` | Page background, card surfaces, form backgrounds |
| Secondary (30%) | `#FFFCF4` | warm white — matches admin sidebar bg | CFI nav sidebar, page shell header card, table row zebra |
| Accent (10%) | `#FFBF00` / `#FFF3C9` | `golden` / amber tints | See reserved list below |
| Muted surface | `#F5F5F5` | `--muted: 0 0% 96%` | Empty state backgrounds, skeleton loading fills |
| Destructive | `hsl(0 84% 60%)` | `--destructive` | Destructive action buttons only |
| Border chrome | `#E9D7A5` | amber border — matches admin | CFI nav sidebar border, active nav item border |
| Dark text | `#0B0B0B` | `darkText` | All primary text |
| Muted text | `hsl(0 0% 45%)` | `--muted-foreground` | Secondary text, placeholders, disabled states |

**Accent reserved for:**
1. Active nav item background in CFI sidebar (`bg-[#FFF3C9]` / border `#D7B24A`) — matches admin pattern
2. CFI zone eyebrow label color (`text-[#9A7A17]`) — "CFI Workspace" label above h1
3. Scrollbar thumb (inherited from globals.css — do not override)
4. Text selection highlight (inherited from globals.css — do not override)
5. CFI sidebar toggle button background and border
6. Log Hours and Log Endorsement primary action buttons (use `bg-golden` / `text-darkText`, not the default shadcn primary which is near-black)

Accent is NOT used for: generic links, table row hover, info badges, read-only data values.

## Layout Pattern

**Page shell:** Create a `CFIPageShell` component — do NOT reuse `AdminPageShell` directly. `CFIPageShell` follows the same structural signature (title, description, backLinks, actions, children) but changes the eyebrow label from "Admin Workspace" to "CFI Workspace". This allows admin users accessing `/cfi` to see the correct zone context.

**Navigation:** Create `CFITopNav` — a separate nav component from `AdminTopNav`. CFI nav sections:
- Schedule (upcoming lessons)
- My Students (roster)
- Availability (weekly template editor)
- Flight Log (log hours for a student)
- Endorsements (log endorsements/milestones)

Admin users who also access `/cfi` should see a cross-link "Back to Admin" in the CFI nav (same pattern as "Back to Admin Dashboard" link in `AdminTopNav`).

**Route segments vs tabs:** The CFI zone uses separate Next.js route segments, not tab state:
- `/cfi` — dashboard / landing (upcoming schedule summary + quick actions)
- `/cfi/students` — student roster
- `/cfi/availability` — weekly availability editor
- `/cfi/log` — flight hour and endorsement logging (single page, two sections or a tabbed sub-view)

Source: Claude's Discretion (CONTEXT.md) — route segments chosen over tab state for deep-linkability and server component compatibility with `requireCFI()` layout guard.

## Interaction States

Each interactive surface must implement all states listed:

### Buttons
- Default: amber accent (`bg-golden text-darkText`) for primary actions; `ghost` variant for secondary
- Hover: `hover:bg-[#FFE79A]` for primary (amber lighten — matches admin toggle pattern)
- Loading: replace label with spinner + "Saving…" / "Logging…" text; disable pointer events
- Disabled: `opacity-50 cursor-not-allowed`

### Table rows (student roster, schedule)
- Default: white background
- Hover: `hover:bg-[#FFFDF7]` (warm white — matches admin nav item hover)
- Selected (if applicable): `bg-[#FFF3C9]` (amber tint)

### Nav links (CFI sidebar)
- Inactive: `border-[#ECE7D6] bg-white text-slate-700`
- Active: `border-[#D7B24A] bg-[#FFF3C9] text-darkText shadow-sm`
- Hover (inactive): `border-[#DCC56F] bg-[#FFFDF7]`

### Form inputs
- Default: shadcn `Input` defaults (border `--input`, ring `--ring`)
- Focus: `ring-2 ring-golden/40` — overrides default neutral ring to match brand
- Error: `border-destructive` + error message below in `text-sm text-destructive`
- Disabled: `opacity-50 cursor-not-allowed bg-muted`

### Skeleton loading
- Use `Skeleton` component with `bg-muted animate-pulse`
- Schedule: 5 skeleton rows at 48px height
- Roster: 6 skeleton rows at 48px height
- Availability grid: 7 × 3 skeleton cells

| Surface | Heading | Body |

| Error | Message | Recovery path |
|-------|---------|---------------|

## Accessibility Requirements

- All interactive elements have visible focus rings (2px golden/40 for primary controls; default ring for inputs)
- All form fields have associated `<label>` elements (use shadcn `Label` with `htmlFor`)
- All icon-only buttons have `aria-label` (e.g., sidebar toggle: `aria-label="Show CFI menu"`)
- Tables use `<thead>` with `scope="col"` on all `<th>` elements
- Dialogs use `DialogContent` with `DialogHeader` and explicit `aria-labelledby` pointing to the dialog title
- Empty state regions use `role="status"` and `aria-live="polite"` so screen readers announce when data loads
- Loading skeletons use `aria-busy="true"` on their container while loading
- Minimum contrast: body text (#0B0B0B on #FFFFFF) = 21:1 — passes WCAG AA and AAA; muted text (hsl 0 0% 45% ≈ #737373 on #FFFFFF) = 4.6:1 — passes WCAG AA
