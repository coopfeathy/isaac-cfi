**`app/api/cfi/schedule/route.ts` — GET endpoint (per D-04, D-05, D-06, CFI-02):**
Use the two-step query approach (safer than nested PostgREST join per RESEARCH Pitfall 5):

1. Import `requireCFI` from `@/lib/auth` — call it with the request. If it returns `error`, return that response.
2. Get the authenticated user from the result (`{ user, profile }`).
3. Query `students` table: `SELECT user_id, full_name, email FROM students WHERE instructor_id = user.id` using `getSupabaseAdmin()`.
4. If no students found, return `[]` with 200.
5. Extract `studentUserIds = students.map(s => s.user_id)`.
6. Compute date window: `today` (start of day in UTC) and `sevenDaysLater` (7 days from today).
7. Query `bookings` with joined `slots`: `SELECT bookings.id, bookings.status, bookings.notes, bookings.user_id, bookings.created_at, slots(start_time, end_time, type) FROM bookings WHERE user_id IN studentUserIds AND status IN ('pending', 'confirmed', 'completed') AND slots.start_time >= today AND slots.start_time <= sevenDaysLater ORDER BY slots.start_time ASC`.
8. Attach student name to each booking from the students list (match on `user_id`).
9. Return the enriched array as JSON.

Use `getSupabaseAdmin()` for both queries (service role bypasses RLS — the route itself enforces scoping via the `instructor_id` filter on students).
