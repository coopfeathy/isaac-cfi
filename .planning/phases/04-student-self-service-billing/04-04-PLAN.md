2. Create `app/api/admin/billing/invoice/route.ts`:
   - Import `requireAdmin` from `@/lib/auth`, `getSupabaseAdmin` from `@/lib/supabase-admin`.
   - Instantiate Stripe: `new Stripe(process.env.STRIPE_SECRET_KEY!, { apiVersion: '2022-11-15' })`.
   - POST handler accepts JSON body: `{ bookingId: string, studentId: string, amountCents: number, description: string }`.
   - Auth: `const adminCheck = await requireAdmin(request); if ('error' in adminCheck) return adminCheck.error;`
   - Validate: all four fields required. `amountCents` must be a positive integer. `description` must be non-empty string.
   - Look up student: `supabaseAdmin.from('students').select('stripe_customer_id, email, full_name').eq('id', studentId).single()`.
   - If no `stripe_customer_id`: return 400 `{ error: 'Student has no Stripe customer. They must save a payment method first.' }`.
   - Create invoice:
     ```typescript
     const invoice = await stripe.invoices.create({
       customer: student.stripe_customer_id,
       collection_method: 'send_invoice',
       days_until_due: 30,
       auto_advance: false,
       metadata: { bookingId, studentId, type: 'lesson_completion' },
     })

     await stripe.invoiceItems.create({
       customer: student.stripe_customer_id,
       invoice: invoice.id,
       amount: amountCents,
       currency: 'usd',
       description,
     })

     const finalized = await stripe.invoices.finalizeInvoice(invoice.id)
     await stripe.invoices.sendInvoice(finalized.id)
     ```
   - Update booking with invoice ID: `supabaseAdmin.from('bookings').update({ stripe_invoice_id: finalized.id }).eq('id', bookingId)`.
   - Return `{ invoice: { id: finalized.id, hosted_invoice_url: finalized.hosted_invoice_url, status: finalized.status } }` with 201.

3. Add `invoice.paid` handler to `app/api/stripe-webhook/route.ts`:
   - Add to the `getEventContext` function: handle `invoice.paid` to extract `bookingId` from `invoice.metadata?.bookingId`.
   - After the existing `payment_intent.payment_failed` handler block (before the final `processed` update), add:
     ```typescript
     if (event.type === 'invoice.paid') {
       const invoice = event.data.object as Stripe.Invoice
       const invoiceBookingId = invoice.metadata?.bookingId

       if (invoiceBookingId) {
         await supabaseAdmin
           .from('bookings')
           .update({ status: 'paid' })
           .eq('id', invoiceBookingId)
           .in('status', ['confirmed', 'pending_approval'])
       }
     }
     ```
   - The existing idempotency check (stripe_webhook_events table) already protects against duplicate processing. No additional dedup needed.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-17 | Spoofing | /api/admin/billing/invoice | mitigate | requireAdmin() validates admin role via Bearer token + profiles.is_admin check |
| T-04-18 | Tampering | invoice amount | mitigate | amountCents validated as positive integer server-side; admin is trusted party |
| T-04-19 | Tampering | webhook replay | mitigate | stripe_webhook_events idempotency table prevents duplicate processing; .in('status', [...]) guard prevents re-updating already-paid bookings |
| T-04-20 | Repudiation | invoice creation | accept | Stripe maintains full audit trail of all invoice operations |
| T-04-21 | Information Disclosure | cancellation_fee_flags | mitigate | RLS allows only admin and service_role access; student cannot read flags |
| T-04-22 | Denial of Service | webhook handler | accept | Stripe retries on 5xx; existing error handling returns 500 and logs; no amplification risk |
</threat_model>
